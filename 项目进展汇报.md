# 🚗 BEVFusion Parking Lot Perception System

A **pure PyTorch reimplementation of BEVFusion** designed to run on **Mac (Apple Silicon, no CUDA)**, with a focus on **parking lot perception and occupancy mapping**.

---

## ✨ Highlights

- 🔥 **CUDA-free BEVFusion** — Runs on Mac (MPS acceleration supported)
- 📷 + 🧭 **Multi-modal fusion** — Camera (6 views) + LiDAR
- 🗺️ **OGM (Occupancy Grid Map)** — Ready for path planning (no training required)
- ⚡ **End-to-end pipeline** — Data → BEV → OGM → Visualization
- 🧩 Modular design — Easy to extend for custom datasets

---

## 🎯 Project Goal

This project aims to reproduce the core BEVFusion pipeline under a **non-CUDA environment**, achieving:

- **Input**: LiDAR point clouds + multi-camera images  
- **Output**:
  - BEV (Bird’s Eye View) semantic map  
  - OGM (Occupancy Grid Map)  
- **Application**: Obstacle perception & path planning in parking lots  

---

## 🧠 Method Overview

### 🚫 Limitations of Original BEVFusion

The original BEVFusion depends on CUDA-specific ops:
- spconv (sparse convolution)
- bev_pool

❌ Cannot run on Mac / CPU-only environments  

---

### ✅ Our Solution (Pure PyTorch)

| Module | Original | Ours | Notes |
|--------|----------|------|------|
| Camera Backbone | ResNet-50 | ResNet-50 | torchvision pretrained |
| View Transform | LSS + CUDA | **LSS + scatter_add** | CUDA-free |
| LiDAR Encoder | VoxelNet + spconv | **PointPillars** | pure PyTorch |
| BEV Fusion | ConvFuser | ConvFuser | unchanged |
| Output Head | TransFusion | **BEV Seg + OGM** | extended |

# BEVFusion 停车场感知系统 — 项目进展汇报

## 项目目标

在 Mac 笔记本（无 CUDA）上复现 BEVFusion 核心流程，实现：
- 输入：LiDAR 点云 + 多路摄像头图像
- 输出：BEV 鸟瞰图 / OGM 占用栅格图
- 应用场景：停车场内障碍物感知与路径规划

---

## 技术方案

### 原版 BEVFusion 的问题
原版 BEVFusion（MIT HAN Lab）依赖大量 CUDA 专用算子（spconv 稀疏卷积、bev_pool 等），无法在 Mac 上运行。

### 我们的解决方案：纯 PyTorch 重写
| 模块 | 原版 | 本方案 | 说明 |
|------|------|--------|------|
| Camera Backbone | ResNet-50 | ResNet-50 (torchvision) | 完全一致，使用预训练权重 |
| View Transform | LSS + CUDA bev_pool | **LSS + scatter_add** | 核心算法一致，去掉 CUDA 依赖 |
| LiDAR Encoder | VoxelNet + spconv | **PointPillars** | 替换稀疏卷积，纯 PyTorch 实现 |
| BEV Fusion | ConvFuser | ConvFuser | 完全一致 |
| 输出头 | TransFusion | BEV Seg Head + OGM | 增加 OGM 输出 |

### 系统架构

```
摄像头图像 (6路)              LiDAR 点云
      ↓                           ↓
 ResNet-50 Backbone          PointPillars
      ↓                           ↓
 FPN Neck                    Pillar Scatter
      ↓                           ↓
 LSS View Transform          LiDAR BEV 特征
 (2D → 3D → BEV投影)
      ↓                           ↓
 Camera BEV 特征 ←——融合——→ LiDAR BEV 特征
                    ↓
              BEV 分割头
                    ↓
         ┌──────────┴──────────┐
      语义 BEV 图           OGM 占用栅格图
   (车辆/行人/障碍物)      (占用/空闲/概率)
```

## 当前完成情况

### 已完成

**1. 开发环境**
- Python 3.10 conda 环境（bevfusion）
- PyTorch 2.11，支持 Apple MPS 加速
- 所有依赖包安装完毕

**2. 核心模型代码（共 7 个模块）**
- `config.py` — 统一配置（BEV 范围、分辨率、类别等）
- `models/camera_encoder.py` — ResNet-50 + FPN + LSS 视角变换
- `models/lidar_encoder.py` — PointPillars 编码器（纯 PyTorch）
- `models/fusion.py` — Camera + LiDAR BEV 特征融合
- `models/heads.py` — BEV 语义分割头
- `models/bevfusion.py` — 主模型（组装所有模块）

**3. 数据处理**
- `data/nuscenes_loader.py` — nuScenes 数据加载（6路摄像头 + LiDAR + 标定参数）
- `data/bev_gt.py` — BEV 地面真值生成（从 3D 标注框投影到 BEV 网格）
- nuScenes mini 数据集已下载（10 个场景，404 个样本）

**4. OGM 生成（已验证可用）**
- `utils/ogm.py` — 三种 OGM 生成方式：
  - 二值 OGM（直接 LiDAR 投影）
  - 概率 OGM（基于点密度）
  - 高度图（每格最大高度）
- `run_ogm.py` — OGM 生成主脚本，已在真实数据上验证

**5. 推理脚本**
- `run_inference.py` — 完整 pipeline 推理，已验证端到端可跑通

### 进行中

## 已验证的输出结果

### OGM 占用栅格图（直接 LiDAR 投影，无需训练）
- 覆盖范围：60m × 60m（可调，停车场建议 40m × 40m）
- 分辨率：0.4m/格（可调至 0.2m）
- 网格大小：150 × 150
- 典型占用率：10-12%（道路两侧建筑、车辆、障碍物）
- 输出格式：PNG 图像 + NumPy 数组（.npy，可直接用于规划算法）

### 输出文件示例
```
ogm_output/
├── overview_000.png       # 完整概览（6路摄像头 + LiDAR俯视 + OGM）
├── ogm_sample_000.png     # OGM详细图（概率图 + 点数热力图 + 高度图）
├── ogm_binary_000.png     # 二值OGM（黑=占用，白=空闲）
├── last_ogm_soft.npy      # 概率OGM数组（下游规划使用）
├── last_ogm_binary.npy    # 二值OGM数组
└── last_height_map.npy    # 高度图数组
```
